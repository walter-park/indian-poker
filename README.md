## 📖 인디언 포커란?

인디언 포커는 자신의 카드를 볼 수 없고, **상대방의 카드만 볼 수 있는** 심리 카드 게임입니다.
상대의 표정과 베팅 패턴을 읽어 승부를 결정하는 것이 핵심입니다.

### 기본 규칙

1. 각 플레이어에게 **1~10 사이의 카드** 1장이 배분됩니다
2. **내 카드는 내가 볼 수 없고**, 상대방의 카드만 볼 수 있습니다
3. 상대 카드를 보고 **콜(Call) / 레이즈(Raise) / 다이(Fold)** 를 선택합니다
4. 숫자가 높은 카드를 가진 플레이어가 승리합니다
5. 상대방의 칩을 모두 빼앗으면 최종 승리!

## 🚀 시작하기

### 방법 1: GitHub Pages (권장)

추가 설치 없이 바로 플레이할 수 있습니다.

1. **[https://walter-park.github.io/indian-poker/](https://walter-park.github.io/indian-poker/)** 에 접속
2. 한 명이 **"방 만들기"** 클릭
3. 다른 한 명이 같은 URL에 접속 후 **"참가하기"** 클릭
4. QR 코드를 스캔하거나, 화면에 표시된 방 ID를 직접 입력

### 방법 2: 로컬 실행

```bash
# 저장소 클론
git clone https://github.com/walter-park/indian-poker.git
cd indian-poker

# 로컬 서버 실행 (Python 3)
python3 -m http.server 8080

# 브라우저에서 접속
# http://localhost:8080
```

> **참고:** QR 스캔(카메라)은 HTTPS 또는 localhost에서만 동작합니다.
> 같은 기기에서 브라우저 탭 2개로 테스트할 수 있습니다.

## 🎮 플레이 가이드

### 1단계: 방 만들기 (Host)

- 닉네임을 입력하고 **"방 만들기"** 버튼을 누릅니다
- QR 코드와 방 ID가 화면에 표시됩니다
- 상대방이 연결할 때까지 대기합니다

### 2단계: 방 참가 (Guest)

- 닉네임을 입력하고 **"참가하기"** 버튼을 누릅니다
- 방장의 QR 코드를 스캔하거나, 방 ID를 직접 입력합니다
- 연결이 성공하면 자동으로 게임 화면으로 전환됩니다

### 3단계: 게임 진행

```
┌─────────────────────────┐
│  상대방 이름    💰 10    │
│       ┌─────┐           │
│       │  7  │  ← 상대 카드 (내가 볼 수 있음)
│       └─────┘           │
│                         │
│     🏆 팟: 2            │
│   라운드 1 - 내 차례     │
│                         │
│       ┌─────┐           │
│       │  ?  │  ← 내 카드 (나는 볼 수 없음)
│       └─────┘           │
│  내 이름        💰 10    │
│                         │
│  [콜]  [레이즈]  [다이]  │
└─────────────────────────┘
```

- **체크 (Check):** 추가 베팅 없이 턴을 넘깁니다 (양쪽 베팅이 같을 때). 양쪽 다 체크하면 카드를 비교합니다
- **콜 (Call):** 상대의 레이즈에 맞추고 카드를 공개합니다
- **레이즈 (Raise):** 베팅을 올립니다 (−/+ 버튼으로 금액 조절). 상대에게 턴이 넘어갑니다
- **다이 (Fold):** 현재 라운드를 포기합니다 (팟을 상대에게 넘김)

### 4단계: 결과 확인

- 양쪽 카드가 공개되고 승패가 결정됩니다
- 높은 숫자 카드를 가진 쪽이 팟의 칩을 모두 가져갑니다
- 같은 숫자면 무승부로 팟을 나눠 가집니다

### 승리 조건

상대방의 칩이 **0개**가 되면 게임 종료! 🏆

### 아키텍처: Host-Client 구조

서버가 없는 P2P 환경에서 게임 상태의 일관성을 보장하기 위해 **Host-Client 모델**을 사용합니다.

```
┌──────────┐                    ┌──────────┐
│   Host   │◄──── PeerJS ─────►│  Guest   │
│ (방장)    │    DataConnection  │ (참가자)  │
├──────────┤                    ├──────────┤
│ • 카드 셔플 │                  │ • 베팅 전송 │
│ • 상태 관리 │                  │ • UI 표시  │
│ • 승패 판정 │                  │           │
└──────────┘                    └──────────┘
```

- **Host:** 카드 덱 관리, 게임 상태 결정, 승패 판정 (심판 역할 겸임)
- **Guest:** Host로부터 상태를 수신하고 베팅 액션을 전송

### 게임 상태 흐름

```
WAITING → DEALING → BETTING → REVEAL → ROUND_END → (반복)
                                                  ↓
                                              GAME_OVER
```

### 메시지 프로토콜

| 메시지 | 방향 | 설명 |
|--------|------|------|
| `PLAYER_INFO` | 양방향 | 닉네임 교환 |
| `CARD_INFO` | Host → Guest | 상대에게 보여줄 카드 번호 |
| `CARDS_DEALT` | Host → Guest | 카드 분배 완료, 베팅 시작 |
| `BET_TURN` | Host → Guest | 베팅 차례 알림 |
| `BET_ACTION` | Guest → Host | 베팅 액션 (call/raise/fold) |
| `POT_UPDATE` | Host → Guest | 팟, 칩 동기화 |
| `ROUND_RESULT` | Host → Guest | 라운드 결과 (양쪽 카드, 승패) |
| `GAME_OVER` | Host → Guest | 게임 종료 |

## 🔧 개발 환경 설정

### 요구 사항

- 모던 웹 브라우저 (Chrome, Safari, Firefox, Edge)
- QR 스캔 시 HTTPS 환경 또는 localhost 필요

### 로컬 개발

```bash
# 간단한 HTTP 서버로 실행
python3 -m http.server 8080

# 또는 Node.js
npx serve .

# 또는 PHP
php -S localhost:8080
```

### 같은 Wi-Fi에서 두 기기 테스트

```bash
# 1. 로컬 IP 확인
ifconfig | grep "inet "

# 2. 해당 IP로 서버 실행
python3 -m http.server 8080 --bind 0.0.0.0

# 3. 다른 기기에서 접속 (예: http://192.168.0.10:8080)
# ⚠️ HTTP이므로 카메라(QR 스캔)는 동작하지 않음 → 수동 ID 입력 사용
```

> **팁:** GitHub Pages 배포 URL을 사용하면 HTTPS가 자동 제공되어 모든 기능이 동작합니다.

## ❓ FAQ

### Q: 인터넷 연결이 필요한가요?

**최초 연결 시에만** 인터넷이 필요합니다. PeerJS Cloud 시그널링 서버를 통해 상대방의 주소를 교환한 후에는 브라우저끼리 직접 통신(WebRTC)하므로, 연결이 수립된 이후에는 인터넷이 끊겨도 같은 네트워크 내에서 플레이할 수 있습니다.

### Q: 몇 명까지 플레이할 수 있나요?

현재 **2명** 전용입니다. 인디언 포커의 기본 규칙에 맞춰 1:1 대전을 지원합니다.

### Q: 카메라가 없으면 어떻게 하나요?

QR 스캔 대신 **방 ID 직접 입력**으로 참가할 수 있습니다. 방장 화면에 표시된 ID(예: `ip-a1b2c3`)를 상대방에게 알려주세요.

### Q: 연결이 자주 끊기면?

- 같은 Wi-Fi에 연결되어 있는지 확인하세요
- 공공 Wi-Fi(카페 등)에서는 P2P 연결이 차단될 수 있습니다
- 모바일 데이터(LTE/5G)로 전환해 보세요
- 연결 실패 시 자동으로 최대 3회 재시도합니다

### Q: 부정행위가 가능한가요?

Host가 게임 상태를 관리하므로, 이론적으로 Host가 브라우저 개발자 도구를 통해 카드 정보를 확인할 수 있습니다. 이 앱은 친구끼리 가볍게 즐기는 용도로 만들어졌으며, 경쟁적인 환경에서는 사용을 권장하지 않습니다.
